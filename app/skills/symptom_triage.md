# Skill: 症状分诊与紧急度评估

## 🎯 Trigger (触发条件)
- 用户描述症状时自动触发（路由为 symptom_analysis 时）
- 用户提到多个症状、严重程度关键词、或持续时间时触发
- 纯知识查询（如"高血压的病因"）不触发

## 📚 Context (背景知识)
- 症状分诊是医疗问诊的第一步，决定后续路径：居家观察 / 建议就诊 / 紧急就医
- 错误分诊可能延误救治（将心梗误判为普通胸痛）或浪费医疗资源（将普通感冒标为紧急）
- 规则引擎可覆盖的常见紧急信号：胸痛+呼吸困难、剧烈头痛+意识改变、大出血等
- 系统已有 clinical_checkpoint 中的症状快照，可作为交叉验证数据源

## ⚙️ Workflow (执行步骤)
1. **症状识别**：从用户输入中提取症状关键词（AC 自动机）
2. **紧急度分级**：
   - 🔴 紧急（立即就医/120）：胸痛+呼吸困难、意识不清、大出血、严重过敏反应、持续高烧>39℃+意识模糊
   - 🟡 建议就诊（48小时内）：症状持续>3天无缓解、反复发作、影响日常生活、用药后无改善
   - 🟢 居家观察：轻微症状、首次出现、无危险信号
3. **症状组合风险检测**：某些症状组合显著提高风险（如头痛+发热+颈僵=脑膜炎高风险）
4. **时间维度评估**：结合 clinical_checkpoint 中的 onset_ts 计算症状持续时间
5. **分诊建议输出**：包含紧急度等级、建议行动、需要观察的信号、何时需复诊

## 📤 Output Format (输出规范)
- 分诊结果包含：紧急度等级（🔴/🟡/🟢）、建议行动、观察信号
- 紧急等级必须包含"如出现X请立即就医"的具体信号描述
- 不给出明确诊断，仅给出分诊建议
- 格式示例：
  【分诊结果】🟡 建议就诊
  【主要症状】头痛、发热
  【持续时间】3天
  【建议行动】建议48小时内就诊，挂号内科或神经内科
  【观察信号】如出现剧烈头痛加重、意识模糊、持续呕吐，请立即就医

## 🛡️ Guardrails (安全边界)
- ❌ 禁止给出确定性诊断（分诊≠诊断）
- ❌ 禁止将紧急症状降级为"居家观察"
- ❌ 禁止替代医生判断用药方案
- ⚠️ 分诊结果仅作为就医参考，不能替代专业医疗评估
- ⚠️ 规则引擎无法覆盖的症状组合，默认建议就诊（宁可过度分诊，不可遗漏危重）
