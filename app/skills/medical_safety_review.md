# Skill: 医疗回答合规与安全审查

## 🎯 Trigger (触发条件)
- 在【答案生成】节点之后、【Redis缓存写入】节点之前自动触发
- 对所有非缓存命中的新生成回答执行强制审查
- 审查耗时上限 800ms，超时则默认 pass 并记录告警日志

## 📚 Context (背景知识)
- 医疗场景中，大模型即使拿到了正确的检索文档，仍可能在生成阶段出现超适应症建议、诊断性断言或语气不当等致命风险
- 本 Skill 作为安全阀门，在答案生成后做结构化后置处理
- 审查维度：诊断性断言、用药安全、紧急风险拦截、免责声明

## ⚙️ Workflow (执行步骤)
1. **诊断性断言检测**：扫描回答是否包含"确诊为"、"就是XX病"、"肯定是"等绝对化诊断表述。若命中，强制替换为"可能提示"、"建议排查"等风险提示句式。
2. **用药安全核查**：提取回答中的药品名称+用法用量，与权威药品知识库交叉校验。若发现超说明书用药、禁忌人群用药或剂量错误，立即标记为高风险。
3. **紧急风险二次拦截**：检查回答中是否遗漏了用户症状快照中的危急重症信号（如胸痛、呼吸困难）。若存在但未给出就医指引，强制追加紧急就医提示。
4. **免责声明注入**：确认回答末尾是否包含标准化医疗免责声明，缺失则自动补全。
5. **审查决策输出**：返回结构化结果 {status: "pass|revise|block", revised_answer: "...", risk_tags: [...]}

## 📤 Output Format (输出规范)
- 仅输出 JSON 格式，不包含任何解释性文字
- status=pass: 直接透传原回答
- status=revise: 使用 revised_answer 替换原回答
- status=block: 丢弃原回答，返回预设的安全拒答模板

## 🛡️ Guardrails (安全边界)
- ❌ 禁止在审查过程中自行生成新的医学建议
- ❌ 禁止修改回答中已正确引用的文献来源编号
- ⚠️ 审查耗时上限 800ms，超时则默认 pass 并记录告警日志
- ⚠️ 规则引擎优先，LLM 仅在规则引擎标记为高风险时介入
